> **Document type:** `reference` | **Diátaxis:** Reference
>
> A Reference Sheet is a lookup document optimized for scanning. It contains
> structured, factual data — configuration parameters, CLI commands, RBAC matrices,
> port inventories, naming conventions, or enumerated values. It does not explain
> concepts or walk through procedures.
>
> Use tables wherever possible. Keep prose minimal. Link to concept and procedure
> documents for context rather than embedding explanation here. If you find yourself
> writing more than two sentences of explanation in a cell, that content belongs
> in a linked overview or pattern.
>
> **Do not use** for step-by-step instructions — use `template_pattern.md` or
> `template_runbook.md`. Do not embed narrative explanation — link to it.
>
> **Suggested location:** `references/`
> **File naming:** `reference_[domain]_[descriptor].md`
